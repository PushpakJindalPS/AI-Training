Data Flow: Passing Data Between Nodes

How nodes talk to each other

Each node outputs JSON data that the next node can read:
// Previous node output
{ "email_body": "Hi, we need a new dashboard...", "sender": "alice@co.com" }

// In the LLM node system prompt, reference it:
Process this email: {{ $json["email_body"] }}

// To reference any previous node by name:
{{ $node["Gmail"].json["subject"] }}
You don't need to memorise this. n8n has a point-and-click expression editor — hover over any field and it shows you the available data from previous nodes.
The expression syntax looks technical but you never have to type it manually — the expression editor surfaces all available fields and you just click to insert. Show this in the live demo to demystify it.

Connecting an LLM to a Workflow

The LLM node is where intelligence enters your workflow

How it works
  1. 1
    Previous node produces data (email, document, form)
  2. 2
    LLM node receives that data + your system prompt
  3. 3
    LLM processes it and returns a result
  4. 4
    Next node uses that result
Setup steps
  • →
    Choose model: OpenAI, Claude, Azure
  • →
    Add API key (done once in credentials)
  • →
    Write system prompt
  • →
    Map input from previous node
Example system prompt inside n8n
You are a professional assistant processing incoming project requests. Read the following input and extract: - Project name - Requestor - Objective - Deadline - Any missing info (open questions) Return as structured JSON. Input: {{ $json["email_body"] }}
This is the direct connection to Session 2 — every prompt engineering skill they learned applies here. The only difference is the input comes from another node instead of being typed manually. Point out the expression — it's how n8n passes data between nodes.

The AI Agent Node

From LLM call to autonomous agent — one node

LLM Node
  • Single call — one input, one output
  • No tools
  • No memory
  • You decide the steps
Agent Node
  • Multi-step — reasons, acts, checks, repeats
  • Can use tools (search, files, email)
  • Can connect to memory store
  • Agent decides its own steps
Configure the Agent node
  1. 1
    System prompt — Role + Goal + Constraints + Output
  2. 2
    Tools — which nodes the agent can trigger
  3. 3
    Memory — optional memory store
  4. 4
    Model — which LLM it uses
The agent loop
Receive goal → Reason: "What do I need first?" → Act: use a tool → Observe: what did it return? → Reason again: "What next?" → Repeat until goal is complete → Return final output
This is the Session 3 content made real — the 5 components (Signal, Input, Reasoning, Actions, Memory) are literally the configuration options in this single node. Point that out explicitly. It's a satisfying moment of "it all connects."

Memory in n8n Agents

Making your agent remember

Window Buffer Memory (short-term)

  • Keeps the last N messages in context
  • Good for conversational agents
  • Built into the Agent node — just enable it
  • Lost when the session ends

External Memory Store (long-term)

  • Stored in a database (Postgres, Redis, Pinecone)
  • Agent retrieves relevant memories by topic
  • Survives across sessions
  • Remembers preferences, facts, history
Practical example
Monday: "Whenever I ask for a project summary, always include the RAG status."
→ Agent stores this preference
Thursday: User asks for a project summary
→ Agent retrieves preference → automatically includes RAG status
Memory is what turns a one-off tool into a genuine assistant. For the live build today, we'll use window buffer memory (simpler). Long-term memory is the next step once participants are comfortable with basic agent building.

Governance & Responsible Use

Using AI tools responsibly in a professional setting

🔒 Data Privacy

  • Know where your data goes when you call an LLM API
  • Enterprise agreements include data privacy protections — consumer versions may not
  • Never send PII or confidential data to a consumer tool without checking your org's policy

👁 Human Oversight

  • Keep humans in the loop for outputs sent externally
  • Use "draft and approve" patterns — agent drafts, human approves
  • Automate low-risk steps; keep judgment on high-stakes ones

📋 Auditability

  • n8n logs every execution — who triggered it, what was processed
  • For regulated workflows, ensure logs are retained
  • Name workflows and nodes clearly so others can understand them

🚀 Start Small

  • Build simple, single-purpose workflows first
  • Test thoroughly before connecting to live systems
  • Add complexity gradually — don't build a 20-step agent on day one
Don't make this slide feel like a legal disclaimer. Frame it as good engineering practice — the same way you'd test a formula in Excel before using it in a board report. The goal is confident, responsible use — not fear.

Test Input

System Prompt in n8n
You are a professional assistant. Read the following project brief and return: 1. A one-paragraph executive summary (max 60 words) 2. A numbered list of key action items 3. A list of open questions or missing information Brief: {{ $json["brief"] }}
What to highlight: The LLM received messy prose and returned structured, usable output. The expression {{ $json["brief"] }} is how n8n passes data between nodes. This took 3 nodes and under 10 minutes to build.

A Full AI Agent

Scenario: An agent that answers questions by searching the web or reading a document — deciding which tool to use based on the question.

Workflow Structure
[1] Chat Trigger → built-in chat UI
↓
[2] AI Agent Node
├── Tool: Web Search (SerpAPI)
├── Tool: Document Reader
└── Memory: Window Buffer (10 msgs)
Test Conversation
Q1: "What is the current interest rate in the UK?"
→ agent uses web search
Q2: "What are the key deliverables in the project document?"
→ agent uses document reader
Q3: "Based on what you just told me, what should be the first priority?"
→ agent uses memory of the conversation
Agent System Prompt
You are a helpful professional assistant. When answering: - If the question requires live or recent information, use the web search tool - If the question is about the provided document, use the document reader tool - Always cite which tool you used - Be concise — answer in 3-5 sentences unless asked for more
What to highlight: Same LLM, same interface. The Agent node decides which tool to use — you only defined the goal and the available tools. This is a working AI agent built in under 15 minutes, with no code.
Pause after each test question and let the audience observe which tool the agent chose and why. The "memory follow-up" question is usually the most impressive — highlight that the agent understood the conversation context without being told.
